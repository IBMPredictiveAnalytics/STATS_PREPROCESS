This procedure applies one or more transformations appropriate for continuous, scale measurement level variables to a set of variables. To some degree, these transformations can be produced by other SPSS Statistics commands, but the versions here have some extra features.

Transformed variables can replace the values in the active file, or they can be written to another dataset. Any missing value definitions and value labels for the variables are cleared in the output dataset, whether the active file or another one, as they would not apply after the transformations, but missing values in the input are converted to system-missing values (SYSMIS) before pprocessing.

Transformations require that all the input cases be help in memory. Weights and split files are not honored. The active file must have a dataset name.

The transformation definitions, which require the calculation of some statistics on the input data, can be saved via the SAVETRANS keyword and applied in a subsequent command using APPLYTRANS. This can be helpful in train/test situations or in any situation where the calculated statistics from the input set of data should be applied to a different set of cases with the same variables.

The Main Command

VARIABLES specifies the variables to be transformed using the STANDARD, RANGE, ROBUST, NONLINEAR, QUANTILE, and KBINS subcommands. If applying previously saved transforms, the variable list can be omitted, but if it is specified for clarity for syntax readers, the names must match exactly the names in the saved transforms file. If multiple tasks are specified they are executed in the subcommand order listed above.

DATASET optionally specifies an output dataset for the transformed variables.

UPDATE specifies whether to update the values in the input dataset. If UPDATE is NO and no dataset is specified, the command would be a no-op.

ID specifies an ID variable. It is required unless APPLYTRANS is used. In that case, it may be omitted, or a different variable may be specified.

SORT specifies whether or not to sort the cases by the ID variable. Cases must be in ascending order of the ID values.

SUFFIX Optionally, a suffix for the transformed variable names can be specified. It will be appended with a leading underscore to the names of the transformed variables. If no suffix is specified and UPDATE is specified, the values of the variables in the input set with be updated. With a suffix, new variables in the updated input will be created, except that if the name with suffix matches a variable in the input dataset, that variable's values will be updated. The suffix should be chosen so as not to create duplicate names.

SAVETRANS optionally specifies a file to hold the transformations specified in the current command. The file extension defaults to pkl. These are not the values of the transformed data but, rather, the transformation methods and parameters estimated in defining the transformation. For example, if standardizing variables based on means and variances, the means and variances of the input variables and the transformation type would be saved along with other parameters.

APPLYTRANS optionally specifies that the definitions and parameter values from a saved transform file are used. Any specifications on the transformation subcommands are ignored. The variable names in the transform file must match the names in the current dataset.

Transformations

Typically, only one transformation would be specified, but if multiple transformations are selected, they are applied in sequence in the subcommand order listed above. Each transformation subcommand has a "DO" parameter that specifies whether to apply that transformation. If it is NO, that transform is not applied regardless of the other subcommand parameters.

The transformations are implemented using the scikit-learn Python modules. See
Scikit-learn: Machine Learning in Python, Pedregosa et al., JMLR 12, pp. 2825-2830, 2011.

The web page for the scikit preprocessing module, which includes functions not part of this extension command, is
https://scikit-learn.org/stable/modules/classes.html#module-sklearn.preprocessing

Scikit-learn is licensed under the BSD 3 license, which is very permissive. A copy is installed with this extension.

/OPTIONS

PRINT specifies whether a table with summary parameters for each transformation should be displayed. If using a transformation file, only the variable names and a short specification will be displayed.

/STANDARD

DOSTD specifies standard scaling for each variable. MEAN and STD specify whether to remove the mean and scale by the standard deviation.

/RANGE

DORANGE specifies whether to scale the variables to a range specified by MIN and MAX. The transformation is
X_std = (X - X.min) / (X.max - X.min)
X_scaled = X_std * (maximum - minimum) + minimum
where minimum and maximum are the specified values.

/ROBUST

DOROBUST specfies whether to do robust scaling by median removal, CENTER, and scaling based on the quantile range, SCALE.

QRANGE species two percentiles for the scale. The defaults are 25 and 75.

Robust scaling may be particularly useful if the data contain many outliers.

/NONLINEAR

DONONLINEAR specifies whether to do a tranformation as close as possible to a normal distribution.

METHOD specifies whether to use the Yeo-Johnson transformation, YEOJOHNSON, or Box-Cox, BOXCOX. Box-Cox requires strictly positive data. The optimal transformation parameter, Æ›, is estimated by the maximum likelihood method. The formulas are below.

STANDARDIZE specifies whether to standardize the resulting variable.

/QUANTILE

DOQUANTILE specifies whether to do a quantile transformation.

"This method transforms the features to follow a uniform or a normal distribution. Therefore, for a given variable, this transformation tends to spread out the most frequent values. It also reduces the impact of (marginal) outliers: this is therefore a robust preprocessing scheme.

First an estimate of the cumulative distribution function of a variable is used to map the original values to a uniform distribution. The obtained values are then mapped to the desired output distribution using the associated quantile function. Variable values of new/unseen data that fall below or above the fitted range will be mapped to the bounds of the output distribution. Note that this transform is non-linear. It may distort linear correlations between variables measured at the same scale but renders variables measured at different scales more directly comparable."

NQUANTILES specifies how many quantiles to use to discretize the CDF.

/KBINS

DOKBINS specifies whether to bin variables into intervals producing the bin numbers as the values. There are three methods for determining the bins, BINDEF.

  • UNIFORM: Equal width bins
  • QUANTILE: Equal number of points in each bin
  • KMEANS: Values in each bin have the same nearest center of a 1D k-means cluster
NBINS specifies the number of bins. The first and last bin edges are extended to minus and plus infinity, respectively when applied to new data.
